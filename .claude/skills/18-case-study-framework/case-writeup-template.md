# Case Writeup Template

A standard structure for writing up a case study analysis for submission.

## Template

### 1. Executive Summary (1/2 page)

State your recommendation upfront in 3-4 sentences:
- What is the decision?
- What do you recommend?
- What is the key supporting evidence?
- What is the expected outcome?

### 2. Problem Statement (1/2 page)

- What is the core problem or decision the protagonist faces?
- What are the key constraints (time, budget, resources, regulations)?
- What are the stakes?

### 3. Situation Analysis (1-2 pages)

- **Industry analysis**: Use Porter's Five Forces or PESTEL as appropriate
- **Company analysis**: Financial performance, competitive position, key strengths/weaknesses
- **Key data**: Summarize the most relevant financial data from the case

### 4. Alternatives (1 page)

Present 3-4 realistic options. For each:
- Brief description (1-2 sentences)
- Pros (2-3 bullet points)
- Cons (2-3 bullet points)
- Key financial metric (e.g., NPV, ROI, payback)

### 5. Recommendation (1-2 pages)

- Which alternative do you recommend and why?
- Financial analysis supporting the recommendation (DCF, multiples, or other quantitative evidence)
- Implementation plan: What steps should be taken? In what order? By whom?
- Risks and mitigation: What could go wrong? How would you monitor and respond?

### 6. Appendices

- Detailed financial calculations
- Sensitivity analysis
- Supporting data tables

## Formatting Tips

- **Use headings and subheadings**: Make it easy for the reader to navigate
- **Lead with conclusions**: In each section, state the takeaway first, then provide evidence
- **Use tables for comparisons**: Side-by-side alternatives are easier to compare than paragraphs
- **Reference case exhibits**: "As shown in Exhibit 3, revenue grew at 12% CAGR from 2018-2023"
- **Be concise**: Typical writeups are 4-6 pages (excluding appendices)

## Grading Criteria (Typical)

| Criterion | Weight | What Professors Look For |
|-----------|--------|-------------------------|
| Problem identification | 15% | Correct identification of the core issue (not symptoms) |
| Analysis quality | 30% | Rigorous use of frameworks and quantitative analysis |
| Use of case data | 15% | Specific references to data in the case, not generic statements |
| Recommendation quality | 25% | Clear, specific, actionable, supported by analysis |
| Communication | 15% | Clear writing, logical structure, professional presentation |
